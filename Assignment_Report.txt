Module 2 : Upload Files
Code :
```dart
class UploadScreen extends StatefulWidget {
  // ...
  Future<void> _uploadAndSubmitOrder() async {
    if (_selectedFiles.isEmpty) return;

    try {
      // 1. Upload files to Supabase Storage
      final fileUrls = <String>[];
      for (final file in _selectedFiles) {
        final filePath = '${userId}/${timestamp}_${file.name}';
        await supabase.storage.from('print-files').uploadBinary(filePath, file.bytes!);
        fileUrls.add(supabase.storage.from('print-files').getPublicUrl(filePath));
      }

      // 2. Create Order Record in Database
      await OrderService().addOrder(
        fileNames: _selectedFiles.map((f) => f.name).toList(),
        fileUrls: fileUrls,
        copies: _copies,
        isColor: _isColorPrint,
        totalAmount: _calculateEstimate(),
      );

      _showOrderConfirmationDialog();
    } catch (e) {
      // Handle error
    }
  }

  int _calculateEstimate() {
    int basePrice = _isColorPrint ? 10 : 2;
    return basePrice * _selectedFiles.length * _copies;
  }
}
```
Explanation :
This module handles the core functionality of the student app. Users can select multiple files (PDF, DOCX, Images) from their device. The app calculates an estimated cost based on print preferences (Color/B&W, Copies). When submitted, files are securely uploaded to a specific bucket in Supabase Storage, and the order details are saved to the database.

Output :
A file selection screen where users can add documents, toggle print settings (e.g., "Color Print", "Double Sided"), see a live price estimate (e.g., "₹20"), and tap "Submit Order".

Explanation :
The Orders screen provides real-time tracking of print jobs. It uses a `TabController` to separate 'Active' orders from 'History'. Output :
A list view of orders. Active orders display a visual progress stepper. Completed orders are moved to the history tab. Tapping an order reveals detailed information like the pickup time slot.

```javascript
export default function Orders() {
    // React State for orders
    const [orders, setOrders] = useState([])
    
    // Subscribe to Supabase Realtime Channels
    useEffect(() => {
        const channel = supabase
            .channel('admin-orders')
            .on('postgres_changes', { event: '*', schema: 'public', table: 'orders' }, 
                () => fetchOrders() // Refresh list on any change
            )
            .subscribe()
    }, [])

    const updateStatus = async (id, status) => {
        await supabase.from('orders').update({ status }).eq('id', id)
    }

    return (
        <div className="admin-dashboard">
            <h1>Print Queue</h1>
            <div className="orders-grid">
                {orders.map(order => (
                    <div className="order-card">
                        <h3>{order.student.name}</h3>
                         {/* File Download Links */}
                        <div className="files">
                            {order.files.map(f => <a href={f.url} download>{f.name}</a>)}
                        </div>
                        {/* Status Action Buttons */}
                        <div className="actions">
                            <button onClick={() => updateStatus(order.id, 'processing')}>Process</button>
                            <button onClick={() => updateStatus(order.id, 'ready')}>Mark Ready</button>
                        </div>
                    </div>
                ))}
            </div>
        </div>
    )
}
```
Explanation :
The Admin Portal is a web-based dashboard built with React. It provides shop operators with a live view of the print queue. Operators can download the files attached to an order, view student details, and update the status of the job. Access is restricted to authorized credentials. The changes made here trigger notifications/updates on the student's mobile app immediately.

Output :
A desktop-optimized dashboard table. New orders pop in automatically. Admins can click "Process" to move an order to the active state, or "Mark Ready" to notify the student for pickup.
